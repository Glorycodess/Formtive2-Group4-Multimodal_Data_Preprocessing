## Quick Start

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

For **webcam support**:
```bash
pip install opencv-python
```

For **microphone recording**:
```bash
pip install sounddevice
# macOS:  brew install portaudio
# Ubuntu: sudo apt-get install libportaudio2
```

---

### 2. Extract features (once, before first run)

Place your images and audio in the `data/` folder (structure above), then:

```bash
python feature_utils.py data/
```

This writes:
- `image_features.csv`  — 512-dim ResNet-18 face embeddings
- `audio_features.csv`  — 42-dim MFCC + roll-off + energy vectors

---

### 3. Run the authentication gate

```bash
python auth_gate.py
```

Optional flags:
```bash
python auth_gate.py \
    --image-csv image_features.csv \
    --audio-csv audio_features.csv \
    --face-thresh 0.80 \
    --voice-thresh 0.80
```

## How Verification Works (Approach B)

1. **Feature extraction** — pretrained ResNet-18 (images) / MFCC+roll-off+energy (audio)
2. **Enrolment** — per-person embeddings averaged into one reference vector
3. **Verification** — cosine similarity between input embedding and reference

```
similarity = dot(embedding_input, embedding_stored)   # both L2-normalised
```

- similarity ≥ threshold → **APPROVED**
- similarity < threshold → **DENIED / Unknown**

---

## Simulating an Unauthorised Attempt

When prompted for a face image, provide an image of someone **not** in the system.
The gate will print `✗ FACE NOT RECOGNISED — access denied` and offer to retry.

To test voice rejection, provide audio from a different person to step 2.

---

## Thresholds

| Modality | Default | Notes                                     |
|----------|---------|-------------------------------------------|
| Face     | 0.80    | Lower = more lenient, Higher = stricter   |
| Voice    | 0.80    | Audio features are tighter, higher default|

Tune with `--face-thresh` and `--voice-thresh` flags.
